# IQDB hash storage implementation plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Store iqdb-rs's signature hash on every `Images` row at index time and use it on the existing-image similarity-search query path, removing the runtime dependency on local thumbnail files for `GET /api/v1/images/{image_id}/similar`.

**Architecture:** New nullable `iqdb_hash` column on `images`, captured by `add_to_iqdb_job` from iqdb-rs's `POST /images/{id}` response, queried via iqdb-rs's hash-only `GET /query?h=<hash>` endpoint. Existing file-based path stays as a transitional fallback for rows with `iqdb_hash IS NULL`. Backfill folds into `scripts/populate_iqdb.py`.

**Tech Stack:** Python 3.14, FastAPI, SQLModel, Alembic, MariaDB, ARQ, httpx, pytest. Spec: `docs/plans/2026-05-07-iqdb-hash-storage-design.md`.

```python
async def check_iqdb_similarity_by_hash(
    iqdb_hash: str, *, threshold: float | None = None
) -> list[dict[str, int | float]]:
    """Query IQDB for similar images using a stored signature hash.

    Mirrors check_iqdb_similarity but without the file upload — uses
    iqdb-rs's `GET /query?h=<hash>` path which re-runs the similarity
    search against the in-memory index using the given signature, no
    bytes uploaded.

    Args:
        iqdb_hash: The signature string previously stored in
            images.iqdb_hash (captured by add_to_iqdb_job).
        threshold: Minimum similarity score (0-100), defaults to
            settings.IQDB_SIMILARITY_THRESHOLD.

    Returns:
        List of {image_id, score} dicts above the threshold. Empty list
        on iqdb-rs unavailability.
    """
    if threshold is None:
        threshold = settings.IQDB_SIMILARITY_THRESHOLD

    try:
        iqdb_url = f"http://{settings.IQDB_HOST}:{settings.IQDB_PORT}/query"
        async with httpx.AsyncClient(timeout=5.0) as client:
            response = await client.get(iqdb_url, params={"h": iqdb_hash})

        if response.status_code != 200:
            return []

        results = response.json()
        return [
            {"image_id": r["post_id"], "score": r["score"]}
            for r in results
            if r.get("score", 0) >= threshold
        ]

    except (httpx.RequestError, httpx.TimeoutException, ValueError, KeyError):
        return []
```

```python
async def iter_image_batches(engine, batch_size: int, *, only_missing_hash: bool):
    """Yield batches of (image_id, filename, ext) using keyset pagination."""
    last_id = 0
    while True:
        async with engine.connect() as conn:
            query = (
                select(Images.image_id, Images.filename, Images.ext, Images.iqdb_hash)
                .where(Images.image_id > last_id)
                .order_by(Images.image_id)
                .limit(batch_size)
            )
            if only_missing_hash:
                query = query.where(Images.iqdb_hash.is_(None))
            result = await conn.execute(query)
            rows = result.fetchall()

        if not rows:
            return
        yield rows
        last_id = rows[-1][0]
```

```python
def add_image_to_iqdb(
    client: httpx.Client, image_id: int, thumb_path: Path
) -> tuple[bool, str, str | None]:
    """Add a single image to IQDB.

    Returns:
        tuple: (success: bool, message: str, iqdb_hash: str | None)
    """
    try:
        if not thumb_path.exists():
            return False, f"Thumbnail not found: {thumb_path}", None

        iqdb_url = f"http://{settings.IQDB_HOST}:{settings.IQDB_PORT}/images/{image_id}"

        with open(thumb_path, "rb") as f:
            files = {"file": (thumb_path.name, f, "image/jpeg")}
            response = client.post(iqdb_url, files=files)

        if response.status_code in (200, 201):
            try:
                iqdb_hash = response.json()["hash"]
            except (ValueError, KeyError, TypeError) as e:
                return False, f"hash parse failed: {e}", None
            return True, "OK", iqdb_hash
        return (
            False,
            f"HTTP {response.status_code}: {response.text[:100]}",
            None,
        )

    except httpx.RequestError as e:
        return False, f"Request error: {str(e)[:100]}", None
    except Exception as e:
        return False, f"Error: {str(e)[:100]}", None
```

- [ ] **Step 5.5: Async fan-out per batch**

Replace the `for image_id, filename, ext in batch:` inner loop with `asyncio.gather` of per-image coroutines. The natural shape:

```python
async def _process_one(
    semaphore: asyncio.Semaphore,
    http_client: httpx.Client,
    session_factory,
    image_id: int,
    filename: str,
    ext: str,
) -> tuple[int, str]:
    """Returns (image_id, outcome) where outcome is 'ok' | 'missing' | 'error'."""
    async with semaphore:
        thumb_path = get_thumbnail_path(image_id, filename, ext)
        if not thumb_path.exists():
            return image_id, "missing"

        # add_image_to_iqdb is sync (uses sync httpx.Client). Run it in a
        # thread so the event loop doesn't block on file IO + iqdb POST.
        success, message, iqdb_hash = await asyncio.to_thread(
            add_image_to_iqdb, http_client, image_id, thumb_path
        )

        if not success:
            return image_id, f"error:{message}"

        try:
            async with session_factory() as session, session.begin():
                await session.execute(
                    update(Images)
                    .where(Images.image_id == image_id)
                    .values(iqdb_hash=iqdb_hash)
                )
        except Exception as e:
            return image_id, f"db_error:{e}"
        return image_id, "ok"
```

Reset to main before starting Task 5: `git checkout main -- scripts/populate_iqdb.py`. The plan above is written against main's version.
- **`add_to_iqdb_job` is in-process.** It uses sync `httpx.Client` even though the function is `async def`. That's existing project convention (the job runs in arq's thread pool); don't change that.
- **iqdb-rs response shape.** Per the iqdb-rs source (`server/src/routes/images.rs`), the JSON on a 200 from `POST /images/{id}` is exactly `{"post_id": <int>, "hash": "iqdb_…", "signature": {...}}` — no envelope. This is the "untagged" `ApiResponse::Ok(T)` variant; failures emit `{"error": …}` with non-200 status, so our status-code check + try/except on `["hash"]` covers both shapes.
- **Self-exclusion in the route.** iqdb-rs returns the query image itself in its results (its signature is in the index); the existing `[r for r in similar if r["image_id"] != image_id]` line handles both code paths uniformly. Don't move the filter into the service helpers.
